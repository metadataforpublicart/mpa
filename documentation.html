General Fields
Field Name ID
Field Category General
Description A unique identifier assigned to the case study.
Requirement Required
Repeatability No, each identifier should be unique to each case.
Data Type String
Usage Notes String should be an alphanumeric code abiding by IPA standards.
Examples 00045, 0003A
Field Name Title - Primary
Field Category General
Description Name given to the project.
Requirement Required
Repeatability No - only one primary title per project.
Data Type String
Usage Notes -
Examples Teeter Totter Wall
Field Name Title - Alternative
Field Category General
Description An alternative name for the project.
Requirement Optional
Repeatability No
Data Type String
Usage Notes The alternative title can be a colloquial name for how the project is commonly, but unofficially, known. It can also be a title in a different language.
Examples ArtsPark (name by which The ArtsPark at Young Circle/Millenium Springs is commonly referred)
Field Name Creator - Primary
Field Category General
Description Person (such as the artist, architect, designer, etc.), group, or entity responsible for creating the work.
Requirement Required
Repeatability Yes – many cases have more than one creator and all creators should be credited.
Data Type String
Usage Notes Names should be recorded according to artists’ preference first, then follow a ‘Firstname Lastname’ or ‘Singlename’ capitalization standard.
Examples Ronald Rael and Virginia San Fratello (Rael San Fratello); Omar Rios (Colectivo Chopeke)
Field Name Creator - Alternative
Field Category General
Description Records an alternative name/moniker/persona/etc. for a creator that is not captured in the ‘Creator’ field.
Requirement Optional - Only needed if creator has another name that is important to be captured
Repeatability Yes - May be repeated for as many alternative names as are necessary.
Data Type Enum
Usage Notes Name should be formatted to artists’ preferences; may include special characters as needed if those characters are supported.
Examples Rael
Field Name Creator - Makeup
Field Category General
Description Describes the makeup of creator(s) represented by the ‘Creator’ field (are they an individual, a group, etc.)
Requirement Required - All creators should have a type assigned.
Repeatability Yes - Should be repeated for as many creators are listed.
Data Type Enum
Usage Notes Individual, Group, Collective, etc.
Examples Individual
Field Name Creator - Role
Field Category General
Description Denotes the role the creator played in the case study; likely aligns with job/profession title, but does not have to.
Requirement Required - Each listed created should have a recorded role.
Repeatability Yes - Should be repeated for as many creators are listed.
Data Type Enum
Usage Notes Artist, Architect, Curator, etc.
Examples Artist
Field Name Funding - Type
Field Category General
Description The funding model or method of funding for the project.
Requirement Required, even if the answer is 'Unknown'
Repeatability Yes - if the funding for a project has multiple types, then the field can be repeated for each.
Data Type Enum
Usage Notes Self-funded, grant-funded, percent-for-art program, unknown, etc.
Examples Crowdfunded
Field Name Funding - Budget
Field Category General
Description The amount for the budget that was allocated for the project.
Requirement Optional - Exact budget amount may not be known.
Repeatability Yes - If the funding for a project is split into disparate pieces, field can be repeated as needed.
Data type String
Usage Notes Should include a number and the currency type; approximations are valid.
Examples 900 dollars
Field Name Funding - Source
Field Category General
Description The person or entity who provided the funding.
Requirement Optional - This information may not be readily available depending on the project
Repeatability Yes - if funding comes from multiple sources, please list them all.
Data Type String
Usage Notes -
Examples The National Endowment for the Arts, Americans for the Arts, the Mellon Foundation, etc.
Field Name Location - Region
Field Category General
Description Region of the world the project comes from.
Requirement Required - Each case should have, at the least, a region recorded for it.
Repeatability Yes - while most proejcts will only have one region, some projects may be located in multiple regions simultaneously, or move regions as part of the project itself.
Data Type Enum
Usage Notes North America, Africa, East and Southeast Asia, West, Central and South Asia, Latin America, Eurasia, Oceania
Examples North America
Field Name Location - Country
Field Category General
Description Country (or countries) the project resides in.
Requirement Required - for searchability purposes, each project should have at least one country listed.
Repeatability Yes - like Region, projects may span national borders. As such, field can be repeated.
Data Type String
Usage Notes -
Examples United States; Mexico
Field Name Location - City
Field Category General
Description City (or cities) the project resides in.
Requirement Optional - It may be difficult or irrelevant to name the city in which the project is located.
Repeatability Yes - like other Location fields, projects may vary across city boundaries and each city should be listed.
Data Type String
Usage Notes -
Examples El Paso; Juárez
Field Name Location - Placement
Field Category General
Description General name for location where project is placed/occurs.
Requirement Required - If no term easily applies, default to ‘public space.’
Repeatability No - If present, one descriptive name should be listed.
Data Type Enum
Usage Notes Controlled list of possible placements can be found on the glossary page. If no other placement is identified, default to ‘public space.’
Examples park
Field Name Coordinates - Latitude
Field Category General
Description Latitudinal coordinates of the project.
Requirement Required - for geo-tagging purposes.
Repeatability Not repeatable.
Data Type Decimal
Usage Notes -
Examples 31.766886
Field Name Coordinates - Longitude
Field Category General
Description Longitudinal coordinates of the project.
Requirement Required - for geo-tagging purposes.
Repeatability Not repeatable.
Data Type Decimal
Usage Notes -
Examples -106.45105
Field Name Date - Year Started
Field Category General
Description The year in which work on the project began (this includes planning and design stages).
Requirement Optional - Date may not be known, or may the same as Year Completed.
Repeatability Not repeatable.
Data Type Date (YYYY)
Usage Notes -
Examples 2019
Field Name Date - Year Completed
Field Category General
Description The year in which work on the project was completed (installed, made available for viewing, etc.).
Requirement Optional - This field is highly recommended but in some cases, the completion date may be unknown.
Repeatability Not repeatable.
Data Type Date (YYYY)
Usage Notes -
Examples 2020
Field Name Duration
Field Category General
Description The duration of the project; the length of time the work has/will be completed and made publicly available.
Requirement Required - While may not be known at time of research, should be added and updated as needed.
Repeatability Not repeatable.
Data Type Enum
Usage Notes Permanent, Temporary, Iterative
Examples Temporary
Field Name Duration - Viewability
Field Category General
Description Denotes the viewability of the project. Is the project currently On-View or Off-View?
Requirement Required - While may not be known at time of research, should be added and updated as viewability of project changes.
Repeatability No - Since each case study is recording one project, only one viewability should be needed.
Data Type Boolean
Usage Notes On-View, Off-View
Examples On-View
Field Name Duration - Specifics
Field Category General
Description Specific and/or contextual information about the duration of the project (e.g. specific length, marking a permanent project as ended, etc.)
Requirement Optional - Some projects may not need more specifics regarding Duration.
Repeatability No - Since the field is free text, all information should be recorded in one field.
Data Type Free Text
Usage Notes -
Examples 45 minutes total
Field Name Event
Field Category General
Description Describes the type of event that the project is associated with/a part of.
Requirement Optional - May not be applicable for all projects.
Repeatability Yes - Repeat as needed for types of events project is part of.
Data Type Enum
Usage Notes parade, festival, workshop, etc.
Examples protest
Field Name Description
Field Category General
Description A free-text description of the case study, which can include a description of work and information about the background and/or context of the project.
Requirement Required - Each project should include a description/narrative.
Repeatability Not repeatable
Data Type Free Text
Usage Notes The length of the description is left to the researcher's discretion.
Examples Teeter Totter Wall was a project aiming to enter the socio-political arena in a playful way. The artists engaged in a creative process to find a mechanism for engaging in an angry, growing and hate-filled political discussion with a presentation that brings empathy and playfulness...
Field Name Subject
Field Category General
Description Subject tag(s) that represent major keywords of the case study and/or project (can also be taken from other fields for searchability reasons).
Requirement Required - At least one subject tag is required to promote searchability.
Repeatability Yes - Can be repeated as many times as needed; ideally, each project should have multiple subject tags.
Data Type String
Usage Notes Recommended to be pulled from LCSH.
Examples Public Health, Literacy, Democracy, etc.
Field Name Related - Images
Field Category General
Description Image(s) assocaited with the project.
Requirement Required - At least one image should be attached to the proejct.
Repeatability Yes - Can be repeated up to 20 images.
Data Type Image File
Usage Notes Images should be .jpg format files.
Examples teetertotter_above_view.jpg
Field Name Related - Links
Field Category General
Description Links to related works and/or supplementary information.
Requirement Optional
Repeatability Yes - Can be repeated for as many links as are applicable.
Data Type Link
Usage Notes -
Examples Link to an interview where the artist talks about their purpose for creating the project, commissioning agency website, etc.

Work Fields
Field Name Materiality
Field Category Work
Description Materials used in the creation of the work and/or parts of the work. Can be materials used in the physical construction (e.g. wood, metal, cloth) or materials used in the experience of the work (e.g. video).
Requirement Required
Repeatability Yes - Can be repeated for as many materials are known.
Data Type Enum
Usage Notes Controlled vocabulary of collected terms can be found here.
Examples Plastic, metal, marble, steel
Field Name Work Type
Field Category Work
Description Medium of the work and the genre within which the work can broadly fit.
Requirement Required - While some cases may be difficult to classify, ideally each should have a medium listed.
Repeatability Yes - While most cases will likely only have one, since some cases combine/exist between mediums, field can be repeated as needed.
Data Type Enum
Usage Notes Controlled vocabulary of collected terms can be found here.
Examples Sculpture
Field Name Dimensions - Amount
Field Category Work
Description Number value listed in the dimensions of the work.
Requirement Optional - Not every project will have measurable dimensions.
Repeatability Yes - Field can be repeated as many times as needed.
Data Type Integer, Decimal
Usage Notes -
Examples 900
Field Name Dimensions - Units
Field Category Work
Description Units for the Dimensions - Amount field listed above
Requirement Optional - Only required if an Amount field is listed.
Repeatability Yes - If more than one Amount is listed, then a corresponding number of units should be listed.
Data Type String
Usage Notes -
Examples dollars
Field Name Dimensions - Summary
Field Category Work
Description Summarizes all of the units associated with the project into one field, including dimensions of project pieces if multiple are present.
Requirement Optional - Required if at least one set of dimensions are recorded.
Repeatability No - Due to nature field, all iterations of dimensions should be combined into one Summary field
Data Type Free Text
Usage Notes -
Examples 900 dollars, 25 meters…
Field Name Theme
Field Category Work
Description Major themes present in the work. This includes themes present within and represented by the work. These themes are different from social/community impact themes that are represented in the IAPA/Impact fields.
Requirement Optional - Not all cases may have an immediately identifiable theme.
Repeatability Yes - Multiple themes will likely be present in most cases, so field can be repeated as needed.
Data Type String
Usage Notes -
Examples play, activism
Field Name Style
Field Category Work
Description Style of the work. For example, if the work is sculptural, what style of sculpture is used, etc.
Requirement Optional - Highly recommended to add more specificity/searchability, but can be omitted.
Repeatability Yes - If multiple styles are identified, the field can be repeated as needed.
Data Type String
Usage Notes -
Examples Representation
Field Name Comment
Field Category Work
Description Other comments about the work that the researcher feels are important to mention but are not captured in other fields.
Requirement Optional - Only necessary if extra information is present the researcher feels is important.
Repeatability No - Because of the field’s variable length, only one Comment field should be needed.
Data Type Free Text
Usage Notes -
Examples A total of 3 teeter totters were constructed and had bike horns attached to the seats for added fun.

IAPA/Impact Fields
Field Name Researcher
Field Category IAPA/Impact
Description Name of the researcher responsible for researching the project for the IPA.
Requirement Required - At least one researcher should be listed for each project.
Repeatability Yes - If more than one researcher is credited, the field can be repeated as needed.
Data Type String
Usage Notes Use same name conventions as specified in the Creator - Primary field.
Examples Jen Krava
Field Name Date - Year Nominated
Field Category IAPA/Impact
Description Year the case study was nominated for the International Award for Public Art (IAPA).
Requirement Required - All IAPA case studies will have a nomination year.
Repeatability No
Data Type Date (YYYY)
Usage Notes -
Examples 2021
Field Name Impact - Statement
Field Category IAPA/Impact
Description Statement written by the researcher denoting the social/community/etc. impact that the case study intends to or has had.
Requirement Required - Considering the nature of the IAPA, a statement regarding the impact of a case study should be present.
Repeatability No
Data Type Free Text
Usage Notes -
Examples This project juxtaposes bright colors, youth, play, fun, and joy against policies and mis-beliefs anchored in nationalism, elitism, militarism, and anti-egalitarianism...
Field Name Context - Type
Field Category IAPA/Impact
Description Category of contextual information provided about the work.
Requirement Optional
Repeatability Yes
Data Type Enum
Usage Notes Community, Social, Political, etc.
Examples Community
Field Name Context - Description
Field Category IAPA/Impact
Description Description/narrative of provided contextual information that the researcher has deemed important to the work and/or its impact.
Requirement Optional
Repeatability No
Data Type Free Text
Usage Notes -
Examples Ronald and Virginia had been involved in a number of projects in and around Marfa, Texas...
Field Name Post-IAPA - Juror Notes
Field Category IAPA/Impact
Description Records juror notes from the IAPA judging phase.
Requirement Required - Each case study should have at least one field of juror notes attached.
Repeatability Yes - Can be repeated for as many sets of juror notes there are for each case.
Data Type Free Text
Usage Notes -
Examples N/A
Field Name Post-IAPA - Updates
Field Category IAPA/Impact
Description Records any updates to the case studies following initial IAPA research.
Requirement Optional - May not be applicable/needed for all case studies.
Repeatability No - Because of the variable length of the field, the one Updates field can be added to as new updates are available.
Data Type Free Text
Usage Notes -
Examples A new version of this project was launched in 2023 in Arizona
Field Name Legal - Copyright
Field Category IAPA/Impact
Description Statement regarding copyright of anything regarding the case study or work/related items requires it.
Requirement Optional - Required if copyright statements are required, but can be omitted otherwise.
Repeatability Yes - Can be repeated if multiple statements are needed.
Data Type String
Usage Notes -
Examples Copyright © 2024 Institute for Public Art
Field Name Legal - Rights
Field Category IAPA/Impact
Description Statement regarding rights and ownership.
Requirement Optional - Required if rights statements are required, but can be omitted otherwise.
Repeatability Yes - Can be repeated if multiple statements are needed.
Data Type String
Usage Notes -
Examples Project rights belong to Rael San Fretello
University of Michigan School of Information © 2024
